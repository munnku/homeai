# HomeAI — UI Design Brief

## アプリの一言コンセプト

> 「あれ、どこしまったっけ？」を家族みんなで解決する、AI搭載の家庭物管理アプリ。

---

## プロダクト概要

HomeAIは、家の持ち物・収納場所をデータベース化し、スマホからいつでも「検索・登録・共有」できるPWAアプリ。
最大の特徴は**登録の摩擦のなさ**。カメラで撮るだけでAIが物の名前を認識し、QRコードをスキャンするだけで場所と物を紐づけられる。

---

## ターゲットユーザー

### メインユーザー
- **20〜40代の家族・同居世帯**
- 「片付けたいけど続かない」「物が多くてどこにあるか把握できない」
- スマホはヘビーユーザーだが、新しいアプリの操作に時間をかけたくない

### ユーザーの行動パターン
- **登録する瞬間**：片付け中、引っ越し後、新しい物を買ったとき
- **検索する瞬間**：「あれどこ？」と思った瞬間、急いでいることが多い
- **家族間の会話**：「電池ってどこにあったっけ？」→ アプリを開いて確認して共有

### ペインポイント
- 物の場所を登録するのが面倒で続かない
- メモアプリや写真では検索できない
- 家族間で情報が共有されていない
- 賞味期限を把握できていない（食品ロス）

---

## 世界観・デザイン方向性

### キーワード
**「整頓された暮らし」「落ち着き」「スマートホーム」「頼れるAIアシスタント」**

### ビジュアルトーン
- **ライトモード基調**（明るい雰囲気）
- アクセントカラーは温かみの感じる、かつAIの信頼感も感じさせる色調
- 角丸・柔らかいカード型UI（硬くなりすぎず、生活感と馴染む）
- 情報密度は**中程度**（チラシほど詰め込まない、ノートほど空白でもない）

### UIの雰囲気の参考
- Notionの落ち着いた情報管理感
- Linearのクリーンで高速なUI
- Apple Homeアプリの部屋・デバイス管理の視覚的わかりやすさ

### 絶対に避けること
- 派手なアニメーション、ゲーミックな演出
- 大量のカラーコーディング（識別できない）
- 一覧が「表形式」だけになること（視覚的な豊かさが失われる）

## 主要画面と設計意図

### 1. ホーム画面（間取り図ビュー）
- 部屋を**グリッド配置した間取り図**で表示
- 各部屋カードにアイテム数を表示
- タイルビュー（シンプルなリスト）との切替ボタン
- **検索バー**が最上部に常時表示（テキスト入力でAIチャットにも切替）
- 右下にFAB（＋物を登録）

### 2. 部屋詳細画面
- 部屋の中にある家具・収納・物を階層で表示
- 家具はフォルダのように展開できる
- 賞味期限が近いアイテムは赤/黄でハイライト
- 処分済みは非表示（「処分済みを表示」トグルで確認可能）

### 3. 物の登録フロー（2パターン）
**場所起点（一括）**：QRスキャン or 場所選択 → カメラ連続撮影 → AI認識確認 → 保存
**物起点（単体）**：カメラ撮影 → AI認識 → 場所選択 or QRスキャン → 保存
- AIの認識結果は**大きな文字で候補表示**し、ワンタップで確定できるUX
- テキスト入力にも素早く切替できること

### 4. 物の詳細ページ
- 名前・写真・場所・メモ・メタデータ（賞味期限・数量）を表示・かつ編集できる
- QRコードを表示・ダウンロードするボタン
- 「移動する」「処分する」「編集する」アクション
- 移動履歴（NodeHistory）をタイムラインで表示

### 5. 検索画面
- テキストボックスにインクリメンタルサーチ
- 結果は「物名 ＋ 場所パス（リビング > テレビ棚 > 2段目）」形式
- AIチャットモードに切替すると、チャットUI（バブル形式）に変化

### 6. オンボーディングウィザード（3ステップ）
- Step 1: 世帯名を決める
- Step 2: 部屋をリストで入力（「＋部屋を追加」ボタンで複数）
- Step 3: 最初の1つを登録してみる
- 全体的にウォームで歓迎感のあるトーン

### 7. QRシート印刷（有料）
- チェックボックスで複数選択 → 「QRシートを生成」ボタン
- A4 PDF: 4列×5行のグリッド、各セルにQR + 物名 + 場所パス

## 機能別プラン表示ルール

| 機能 | 無料 | 有料 |
|---|---|---|
| テキスト登録・検索・間取り図 | ✓ | ✓ |
| カメラ→AI認識（1日10回） | ✓ | ✓無制限 |
| 写真保存 | ✗ | ✓ |
| AIチャット | ✗ | ✓ |
| プッシュ通知 | ✗ | ✓ |
| QRシートPDF | ✗ | ✓ |

- 無料プランで有料機能にタッチしたときは**軽いモーダル**でアップグレード誘導（しつこくしない）
- 広告は**バナー形式**で下部に表示（コンテンツを邪魔しない位置）

---

## ブランドパーソナリティ

| 属性 | 内容 |
|---|---|
| 性格 | 頼れる・賢い・押し付けがましくない |
| 話し方 | 丁寧だが簡潔、AIらしい冷静さ |
| 色 | 暖色系で淡い色調のグラデーション |
| 素材感 | すりガラス・金属・マット |

---

## 技術制約（UIに影響するもの）

- **PWA**（ブラウザベース）：ネイティブAPIは一部制限あり
- **モバイルファースト**：基本的にスマホ縦画面で設計。タブレット・PC閲覧も考慮
- **オフライン対応**：最後のデータを表示できる（ローディングスケルトンは必須）
- **間取り図**：グリッドベース（CSS Grid or SVG）で実装予定。複雑な多角形は不要
- **カメラ**：`<input type="file" capture="environment">` または WebRTC
